  3. Node Module Scope!
    node app.js 와 같은 명령어는 app.js 만을 실행합니다.

    다른 파일(util.js)을 함께 실행하기 위해서는 다음의 코드와 같습니다.

    >> node app.js 
     I am utils.js
     I am app.js

    그렇다면 다음의 결과는 어떠할까? 

    >> node app.js
     ERROR

    이 결과는 각각의 .js 파일은 각각의 변수와 스코프를 지니기 때문이다.

    그러나 util.js 의 변수를 app.js 에서도 사용할 수 있게 할 수 있는 방법이 있습니다.

     >> node app.js
     Peter

    app.js 에서 const name 변수의 이름은 util.js 와 독립적이기 때문에 어떻게든 지어도 상관이 없습니다.

    여기서 변수만 export 할 수 있는 것이 아니라 함수도 export 할 수 있습니다!

     >> node app.js
     any Function!

  3. node_module 폴더가 없을 때는, npm install 명령어를 이용하면 다시 npm 모듈이 설치 됩니다.

  4. npm install <package> -g
    -g 표기는 global 하게 설치하는 것입니다. 즉 이 프로젝트에만 적용되는 것이 아니라 OS에 설치하게 됩니다.
    ex)
    npm install nodemon -g
    위의 명령어로 nodemon package 를 OS 에 설치했습니다. MacOS, Linux 의 경우 Sudo 명령어를 이용해야 합니다.
    nodemon package 는 일일히 terminal 에 node app.js 를 칠 필요가 없습니다. nodemon app.js 를 터미널에 치게 된다면 app.js 파일에 변동이 생길 때 마다 자동으로 변경 사항을 적용해서 터미널에 결과를 보이게 합니다. 굉장히 편하게 만들어주는 모듈입니다.
